Good observation — those extra letters (`Ss`, `S+`, etc.) are **modifiers** added to the basic state.

---

## 🧠 How to read process state

The `STAT` column in `ps` is:

```text
<state><extra flags>
```

So:

* First letter → **main state**
* Remaining letters → **additional info**

---

## ✅ Main state (you already know)

| Code | Meaning               |
| ---- | --------------------- |
| `R`  | Running               |
| `S`  | Sleeping              |
| `Z`  | Zombie                |
| `T`  | Stopped               |
| `D`  | Uninterruptible sleep |

---

## 🔍 Extra flags (this is what you're seeing)

### 🔹 `s` (lowercase)

👉 **Session leader**

* Process started a session
* Usually shells or important parent processes

Example:

```text
Ss
```

= Sleeping + session leader

---

### 🔹 `+`

👉 **Foreground process (in terminal)**

* Running in the current terminal
* Gets keyboard input

Example:

```text
S+
```

= Sleeping + foreground process

---

### 🔹 `l` (lowercase L)

👉 **Multi-threaded process**

Example:

```text
Sl
```

---

### 🔹 `<`

👉 **High priority process**

---

### 🔹 `N`

👉 **Low priority (nice value)**

---
